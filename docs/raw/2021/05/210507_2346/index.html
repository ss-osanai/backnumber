　最低限の人生システム、あるいは、
　生活システム、もしくはビジネスシステム、

　またはワークシステムやスタイルと
　いったものを一たび、構築できれば、

　その後は、普通に生きているだけで
　より良くなっていく、

　というか、複利で回っていきます。


　この骨格づくりを
　まず優先してやりましょう、

　そんな話を今日はいたします。


　-----------------------


　よく、システムの話をする時には
　モジュール化という話をいたします。

　これはシステム化音源でも
　お話をしたことだと思います。


　そこに対しての
　復習的な要素もあると思いますが、
　
　結局のところ、システム化音源で
　お伝えしたシステムというのは、


　さまざまなサブシステム、
　すなわち、色々な要素の集合体として、

　いろんな要素が摩擦なく矛盾なく、
　つなぎ合わさることによって結果が出る、

　そんな「系」のこと。


　-----------------------


　しかし、残念ながら、

　我々が思っているほどスキルというのは
　モジュール的なものではありません。

　何かを学び、カチャッとあてはめれば、
　即、結果が出るというものではない、

　ということです。


　-----------------------

　
　なぜかと言うと、

　例えば本の著者であったりとか
　セミナー講師であったり、
　講演家であったりといった人が、

　あなたがこれをやると
　上手くいきますよ、

　ということを色々な形で話を
　されることがありますね


　それはその人にとって最適化されている
　やり方であったり、方法論であって、

　それをそのまま、その人のシステムから
　部分的に切り出してきて、

　そのモジュールを渡して、
　あなたもこのモジュールを入れれば
　同じ結果が出ますよ、

　と言っても、

　肝心のシステムの本体が
　語っている人と自分のものは別物ですから、

　そこに矛盾が生じるのです。


　-----------------------


　製造業的な表現をするならば、

　すり合わせと、
　いった言い方でもいいのかもしれませんが、

　（確かにガチャッとあてはめれば結果が出る、
　　そういう知識もありますが）

　ほとんどの場合、駆動するとか、稼働する、
　結果が出ることを約束されているシステムは、

　知識単体ではなく、さまざまな要素が
　一体化されているものです。


　-----------------------


　と、するならば、
　我々が学ぶときには、

　その一体化されたシステム全体を
　全体として捉えて、

　まるごとインストールするのが
　最も早く、確実なのではないか、

　という話を

　『システム＆カオス音源』

　あるいは

　『題名のない放談会音源（質疑応答編）』

　など、いくつかの音源でも
　話をさせて頂いていたと思いますが、

　そういうことを前提として、
　話を続けます。


　-----------------------


　単発で色々なところから
　さまざまな学びを引っ張ってきても、

　こちらを立てれば
　あちらが立たずということとなり、

　結果として、学びに対して
　迷ってしまう人が多くなるわけです。


　-----------------------


　これでいいのかな、とか、
　昔聞いた話と違う、とか、

　自信を持って学べない、
　距離をおいて学ぶしかない、

　といったような
　没頭・没入できる学びでなく、

　これでいいのかなと迷いつつ学ぶことが
　常態になっている人が少なくない。


　-----------------------


　本能的にせよ、意識的にせよ、

　この学びは今のシステムと
　すり合わせられるのかな、

　という不安であったりとか、
　心配であったりが、

　学びを遠ざけることにつながるのです。


　-----------------------


　このことを理解して、都度の学びを
　きちんとシステムに組み込める人もいれば、

　なかなか組み込めない人もいます。

　これが「学びの歩留まり」と言っても
　良いのではないかと考えています。
　

　-----------------------

　
　我々は無尽蔵に時間を学びに
　投入できるわけではないわけですから、

　いかに単位時間あたりの
　学びの効果というのを最大化できるかが、

　一番の課題になるわけです。


　-----------------------


　皆さんも学びながら、これは血となり
　肉となっているのだろうか、

　と思いながら勉強したり、経験したり、
　見聞きしたり、

　ということもあると思います。


　結局のところ、
　システムに対して思いを致しているのか、

　システムではなく、
　枝葉末節のところに着目しているのか、

　といった、そういう観点から
　学びというのを捉え直されると、

　「学んで学ばず」

　みたいになる、

　あの気持ち悪さの原因が結構、
　見えてくるのではなかろうか、

　という風に思います。


　-----------------------


　僕自身も、色々学んでいる、
　こんなに勉強しているのに、

　なぜ、結果が出ないんだろう、
　作り変わっていく感じがしないんだろう、

　と長年、思っていたわけですが、


　なんのことはない、
　そうなってしまっていた理由は、

　まさにモジュールを大量に集めてきて、
　刷り合わせもなく、

　無理やりくっつけようとしていたから、
　内部に摩擦が生じていたからであり、


　これが原因で、

　いつも懸命に勉強しているけれども、
　一向に学べた気にならない、

　という感じがずっと続くことになっていた
　わけです。


　-----------------------

　
　だからこそ、さっき申し上げたような、

　師匠、先生、あるいはメンターといった、
　ご自身にとって合致したタイプ、

　─また後でその合致というのが、何をもって
　　合致と言うのかについてお伝えしますがー
　
　
　自分がこの人はと思った人に対して、
　そのシステムを丸ごと頂戴する、

　という感覚で接することによって、


　初めてここまで話をしてきたような事が
　この身体を通して理解できる、

　あるいは身体を通して
　学びを取りに行くことができる、

　ということです。


　-----------------------


　結局、

　師匠、先生、メンター、
　呼び方は色々あると思いますが、

　何を学ぶのかというと、
　方法論ではなくて、

　システムそのものを頂戴するのです。
