　結局、

　師匠、先生、メンター、
　呼び方は色々あると思いますが、

　そうした方々から何を学ぶのかというと、

　方法論ではなくて、
　システムそのものを頂戴するのです。


　-----------------------

　
　もちろん、

　ある程度、大きな成果を出しているとか、
　長く生きてこられた方は、

　その分だけシステムが強大であったり、
　システムが精緻であったりとかするので、

　まるまるインストールするのは
　なかなか大変だと思いますが、


　ただしどんな構築物にも
　構造や骨格があるように、

　どんなシステムにも
　構造、骨格はあるはずですから、

　それを抜き出し、把握するのです。


　つまり、

　「何が、この人を
　　この人たらしめているのだろうか」

　「何が、先生を
　　先生たらしめているのであろうか」

　こういう問いを立てるということです。


　-----------------------


　（中略）

　例えば、あなたが営業だとして、
　仮に、営業の能力を高めようと思った時に、

　営業の本を100冊読むのと、

　あなたがこれはと思った先生、師匠に
　一週間同行し、OJTしてもらうのと、

　どちらが営業力が上がるかという問いです。
　

　-----------------------


　営業の本を100冊読むと
　どういうことになるかというと、
　
　究極のところ、皆、言っていることが
　バラバラだと気づく。

　人によって書いている内容が
　まるで異なるわけです。


　著者ごとに異なる価値観や性格や能力があり、
　取り巻く環境があるわけですが、

　各人、自分仕様に最適化させていった、
　ということになりますね。


　-----------------------


　それぞれの人が試行錯誤する中で、
　
　内外の環境に整合性をとり、
　一貫性を作り出せたときが

　「成果を出せる方程式が確立されたとき」

　と言っても良いかもしれませんが、


　そうして確立された一貫性に基づき、

　こうやったら俺は（私は）
　結果を出せたんだ、

　だからあなたもやればいい、
　などと書いてあったりするわけですが、

　できないものはできないですし、
　やりたくないことはやりたくない。

　（＝相性があるということ）


　-----------------------

　
　たった一人の教えにおいてすら、
　そのシステムの構造部分で、

　価値観が合わないとか、
　性格的にできないとか、
　能力的にできない、

　ということがある場合もあります。

　だから相性が大事。


　まして複数の著者の言っていることを
　あれこれとつぎはぎして、

　それでうまくいくはずがないじゃないか、
　ということ。


　-----------------------　


　だから、それぞれの本の良さそうな
　ところをあれこれつまみ食いして、

　数多くのモジュールをくっつけて、
　無理矢理つないで稼動させようとしても、

　（全部がつながれば
　　凄いことになるのでしょうけれども）

　現実には単に複雑さが増すだけで
　混乱や混沌を極めて、

　私たちが前に進むことをかえって
　拒むようになります。
　　

　これが、あれこれ学ぶ過ぎる弊害。

　四方八方に手を出し、学び過ぎている人に
　成果は出せません。


　-----------------------


　ところが、相性の合う、
　たった一人の営業の先生がいて、

　その人と会い、仲良くなって
　一週間、同行するとします。


　もちろん、

　全人格的にであれば理想ですが、
　そうでなくとも、

　すくなくともその人の、ある部分に
　リスペクトできるところがあれば、

　その人を形作っているシステムを
　学びたい、インストールしたい、
　
　という、素直な、ピュアな欲求が
　湧いてくるものです。

　この感情が大事なのです。


　感情の動かぬ学びは

　「学んで学ばず」

　となり、時間のムダ、金のムダ、体力のムダ、
　となるだけです。


　-----------------------


　（中略）
　

　2割8割のパレート法則のうち、

　8割の成果を規定する2割の部分は、
　身体感覚を伴って身につく、

　つまり、営業の肝はこれだ、
　ということが分かるはず。


　それが先ほど「構造」と
　ご説明したものです。


　-----------------------


　コミュニケーションや
　営業、事業開発などにおいて、

　主要な部分は
　あれこれいろんな人から学ぶよりも、

　自分と相性が合う、
　
　（どういう人を選ぶべきかは
　　後に話をいたしますが）

　自分と合致する人から入れたほうが
　早いのです。


　-----------------------
　

　もちろん、私も、学びを開始したときは
　そんなことはわからず、

　さっきお伝えしたように、
　あれこれモジュールを拾ってきては、

　これとこれとこれをうまく組み合わせれば
　いいだろうと考えていました。


　たとえば、

　Aさんが提唱しているこのやり方と
　Bさんが提唱しているやり方と、

　Cさんが提唱しているやり方をくっつけたら、
　これは3倍、あるいは3乗になるのではないか、

　そう思って取り組むわけですが、
　実際は1/3も稼働しません。


　モジュールを寄せ集めても
　システムとして稼働はしない、

　そういうこと。
　

　-----------------------

　
　先に原型を入れてしまうことで、
　取捨選択の基準ができる。

　これができてから新たな学びを行い、
　付け加えていく、

　という順番が効果的。


　-----------------------


　学びのまず最初に

　「この人いいな」

　という人を見つけたら、
　そこに突っ込んで行って

　その後に、多様性を求めて
　色々な人から学んでいく、

　という順序を意識することが大事。


　-----------------------


　結局、人づきあいと切り離された
　（深い）学びというものは存在しない。

　身体と切り離された学びもありません。


　-----------------------


　皆、勉強するとは、
　知識をインプットすること、

　と解釈し、勉強している
　つもりになっているだけです。


　それは受験勉強の弊害であり、

　身体と切り離され、
　遊離したところでの学びであって、

　真に人生、生活仕事において
　意味のある学びというのは、

　自分自身に組み込まれ、
　さらに言うならば、

　習慣と一体化された学びです。


　そこまで持っていけて
　初めて、生きた学びにになるのです。


　-----------------------


　学びには人間が必ず介在します。

　単純に知識をインプットするための
　頭脳だけでなく、人間そのものが介在する。


　とするならば、
　学びそのものもに人間を介在させる、

　すなわち、ある人物を五感で感じることが
　非常に重要である、と思うのです。


　-----------------------


　さきほどお伝えした本を100冊読む、

　というのは、いわば、
　サプリメントのようなものともいえるでしょう。

　確かに、本に書いてあることは
　実際に成果の出せた手法です。


　たしかに、正しいは正しいのですが、

　そこには人間の呼吸であったり、
　鼓動であったり、

　といった人間らしさが抜け落ちます。
　（全てとは言いませんが）
　

　-----------------------

　
　ところが、これと決めた人が出来て、
　しかも、その人が身近に感じられたならば、

　それは「五感で感じる」ということですが、

　これによって学びの歩留まりが
　明らかに高くなっていく、

　ということが間違いなくあるのです。
