■こんにちは、鮒谷です。


　今日は朝から個別コンサル３件、

　夕方から夜にかけては秋葉原のヨドバシカメラにて、
　いろんなものを購入したり、契約したり、解約したり。


　もろもろ完了するのに、
　夜９時過ぎまでかかりましたが、

　これで数ヶ月来の懸案がほぼすべて解決。

　やれやれです。



■さて、

　２日間セミナーにお越し下さった

　株式会社dos（ディーオーエス） 社長の
　山田誠一郎さんより、

　以下のようなご感想を頂戴いたしました。


　（ここから）
　----------------------------------------------------------------


　鮒谷さま、


　2日のセミナー、ありがとうございました。

　セミナー終わったばっかりでまだ頭が
　整理されておらず、
　
　セミナーの書き取りのノート９ページを
　これからまとめる予定です。
　
　
　私がセミナーで一番頭に残ったのは、やはり
　
　「ＯＳ/アプリ/アドオン」
　
　です。
　
　
　私としては
　
　「学ぶことにも重要度のレベルがある」
　
　と解釈しました。
　
　
　やみくもに知識を入れるのではなく、
　それをレベルに分けて理解し、
　
　また重要度、波及効果の高いＯＳレベルを
　先に入れる必要がある、
　
　と解釈しました。
　
　
　随分前のメルマガにその
　
　「ＯＳレベル」
　
　のリストがあったかと思いますので
　記録のノートを読み返します。
　
　
　また、
　
　「自分の教科書編纂」
　
　として、それらをexcelにまとめようかと考えています。
　
　
　また、
　
　「外に出る」
　
　も、私に欠けている気がしました。
　
　
　外へ出て、社会に「掛ける」
　
　＝私は
　
　　「（マンザイの）ネタおろし」
　
　　と命名しましたが、
　
　自分の中に留めず、それを出し、リアクションを得て
　PDCAを回す必要があると感じました。
　
　
　また一番大事だと思っていることが、
　
　「鮒谷さんのセミナーに20万も払って出た、
　　オレスゴイだろ」
　
　で終わらせてはいけない、ということです。
　
　
　過去、セミナーに出たり、本を読んでも、そのことに満足し
　前と変わっていないのでは？と思うことがありました。
　
　日常の引力で引き戻されてしまうのです。
　
　
　よって今回、鮒谷さんの言葉を得て、
　セミナー代以上の価値を作る
　
　（下衆な言葉で言えばそれ以上稼ぐ）

　ことをまずは目標として、今後共精進したいと思います。
　
　このたびはありがとうございました！
　
　
　　（株式会社dos（ディーオーエス） 代表取締役　山田誠一郎 様）

【  学ぶことに熱心なあまり
　　　　　　　　「いろいろ知っている便利屋さん」に陥ることを恐れよ  】

───────────────────────────────────



■私たちは、ついつい

　「なんでもかんでも、学べば学ぶほどよい」

　という錯覚に陥りがちですが、


　「学ぶべきことにも優先順位がある」

　「ときには、学ぶことを思い切って諦める、捨てる」


　ことを意識するのも大切なことだと思います。



■たとえば、私は今日、

　秋葉原のヨドバシカメラに行き、様々な買い物とともに
　IT関連の書籍も数冊買いましたが、


　いつも同様、ここで意識したのは

　「便利アプリ500選」

　とか

　「～を徹底的に深く知り、使い倒せ！」

　みたいなものに絶対に手を出さないこと。



■それらを全部、自分で学び、試していたら、
　どれだけ時間があっても足りません。


　ならば、


　（IT分野ならIT分野において）

　成果を生み出すほんの少数の、
　しかしながら、波及効果の高い、

　そんなものだけを
　しかるべき人に教えてもらったら、
　
　「後は思い切って、捨てればいい
　　（知る必要はないと切って捨てる）」


　と考えているのです。

　

■特にあなたや私が、その分野の専門家でなく、
　深く知る必要がないのであれば。

　そしてほとんどの場合、そうなのです。

　
　学ぶことは目的ではなく、あくまで、
　手に入れたいものを手に入れるための手段、
　
　と両者を明確に峻別しておくべき。



■だから今回、私が買ったのは

　「ダイレクトに生産性向上に直結しそうな
　　ヒントのみが書かれている書籍」

　のみ。


　それをさらっと読んだら、
　あとは、詳しい人にお願いするのです。

　自分では詳しいことは分からないし、
　学ぶ気もないし、その必要もないから。

　結果だけを手に入れられればそれでよし。
　細かいことにはなんの興味もありません。



■ほとんど同じようなことですが、

　「これを知っていたら自慢できる」

　とか

　「知らないよりは知っていたほうが、
　　ほんの少しだけ便利（かも知れない）」

　といったものにも目もくれません。



■こうして捨てるべきものを明確にするから、

　拾うべき（取り組むべき）ことに、
　集中して取り組むことができるようになります。

　
　IT分野に限らず、あらゆる分野において、
　
　「どうでもいいことは、
　　あえて視界に入れない」

　ことを意識しないと、

　
　せいぜい、

　「いろいろ知っている便利屋さん」

　にとどまってしまうことを恐れなければなりません。



■少なくとも私は

　「いろいろ知っている便利屋さん」

　としてよりも、


　「ほとんどのことは何も知らないけれども、
　　ごく一部のことについては圧倒的な知見がある。

　　そしてやるべきこと以外は、上手に、
　　それが得意な人にお願いしてやってもらう。

　　そんなメリハリを効かせられる人」


　として憶えられたいと思っています。
