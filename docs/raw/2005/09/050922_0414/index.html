　なのですが、

■本の置き場の確保と、すべてを読みきる時間がとれないことが
　目下の悩みです。。

＜今日のピークパフォーマンス方程式。＞

【　読書においては速読と精読はスイッチヒッターのごとくに使い分ける。 】

───────────────────────────────────


■情報収集にもいろんな手段があります。

　そのなかで最も古典的、かつ有効な手段のひとつが読書です。


　きょうは、この読書の方法について、私なりの所感を
　述べたいと思います。


■読書法には大きくわけて、

　いわゆる【 速読 】と【 精読 】というものがあります。


■「速読派」といわれる人たちは、おそろしく速いスピードで
　ページをめくっていきます。

　瞬間的にその意味を脳にインストールすることによって、
　効率的に情報収集を図っています。


■そうかと思うと一方では、速読は必要ないとする「精読派」も
　存在します。

　本というものは、ゆっくりと味わい、咀嚼しながら読むもの
　であるというのが精読派の主張です。


■この２つの意見が局地的に論争を呼んだりもしているようですが、

　私の考えとしては、両方とも必要じゃないか、と思っています。


■どんなことにおいても言えることですが、オプションは
　多ければ多いほど良いのです。

　そのほうがシチュエーションにあわせて使い分けることができて
　いいですよね。


■時間がなく、短期間に知識を身につけなければいけないときに
　速読術はきわめて便利なスキルでしょう。


　いっぽう、うまい言い回しや文学的表現を身につけたいとき、あるいは
　非常に手ごわい含蓄のある文章に触れるときには精読もいいものです。


■ひとつの文章に向かって視点を止めて、その文章を出発点とした
　広大な思索の海を自由気ままにたゆとうことができるのは、
　精読ならではなせるわざでしょう。


　気がついたら、ひとつの文章に5分も10分も没入していた。
　あるいはそれ以上長い時間を使って考えていたということが
　あってもいいと思います。


■だいぶまえに、ご紹介したこともあるのですが、
　
　わたしのビジネス人生に多大な影響を与えてくれた本、


【今日のお勧め本　デキる人の法則】
 ⇒ http://amazon.co.jp/o/ASIN/4047914355/2ndstagejp-22/ref=nosim


　において、著者のティム・サンダース氏は、


　「30分以上本を読み続けたら必ず本と目を閉じよう。数分間の間、
　　気持ちを漂わせよう。紙の上では起こりえない形で、頭の中で
　　アイディアがつながるにまかせてみよう」

　と述べ、

　思考しなければいけないときには、本を閉じて考えるという
　ことをしなければならない、

　といったことも述べていました。


■また、逆に、

　知識の断片を圧倒的な短期間に入力する能力は、
　現代人にとって身に付けておいて損はないスキルでしょう。

　損はないというより、これだけ情報があふれている世の中では
　必要不可欠な力といっても過言ではないのかもしれません。


■私もふつうの人の何倍かのスピードで本を読むことはできますし、
　新聞雑誌についても平均的な人よりはすばやく目を通すことができると
　思います。

　そういう意味で、速読も大変重要なスキルであると思っています。
　
　もちろん、わたしもいろんな速読法を学びましたし、現在は
　オリジナルの速読法を必要に応じておこなっています。


■では速読のみを礼賛し、勧めているかというとそうではなく、


　私も本当に素晴らしい名著に接するときには、
　1冊の本に5時間も10時間もかけて、虚心坦懐にページを
　繰（く）ることもあります。


　最近であれば、たとえばこんな本がそうです。


　【今日のお勧め本　
　　　　　　アルビン・トフラーの戦争と平和―21世紀、日本への警鐘】

　 ⇒ http://amazon.co.jp/o/ASIN/4594010938/2ndstagejp-22/ref=nosim


　ワンセンテンスごとに立ち止まり、じっくり向き合い、
　かみ砕きながら読み進んでもなお味わい深い本でした。



■大事なことはその場面その場面に応じてスイッチヒッターの
　ごとく、右か左かいずれのバッターボックスに入るべきか
　選択できる能力だといえるでしょう。


■精読だけが文章を本当に味わうことができる、速読は邪道だとか、

　今の時代に精読なんかしているなんて信じられないとか、

　双方の長所を認めないのではなく、


　相手が右投手だったならば左打席に、

　左投手だったならば右打席へと自由自在にスイッチするように、

　状況に応じて情報収集のしかたを変えられるようになることが重要。


■それによって、いっそうインプット・アウトプットの
　パフォーマンスを高めることができるのではないかと考える
　今日この頃なのです。
